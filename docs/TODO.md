# word_guess_game 핵심 기능 및 TODO

## 1) 프로젝트 기준 개요
- 난이도: ★★☆
- 예상 작업일: 3일
- 권장 주차: Week 1 2일, Week 2 1일
- 핵심 기능: 6회 추측 5자 단어 게임, 일일/무한 모드 분기, 카테고리 모드, 상태 저장, 통계 분석이 핵심인 두뇌게임 앱.

## 2) 2주 실행 계획
### Week 1
- [ ] W1-$((0 + 1)): P0 진행 상태/세이브-리스토어 일관성 점검 및 라우트 전환 이력 테스트
- [ ] W1-$((1 + 1)): P0 입력 제어 경로(터치, 드래그, 버튼) 동시성 정리 및 예외 처리 강화
- [ ] W1-$((2 + 1)): P1 난이도 전환/재시작 규칙 재검토: 시드·타이머·점수 계산의 결정성 확보
### Week 2
- [ ] W2-$((0 + 1)): P1 Undo/Redo 및 승패 판정 회귀 테스트 자동화
- [ ] W2-$((1 + 1)): P2 성능 튜닝: 애니메이션 프레임 안정화 및 빌드 오버헤드 하향

## 3) 현재 버그 가능성 추정
| 위험 영역 | 발생 확률 | 영향도 | 점검 포인트 |
| --- | --- | --- | --- |
| 상태 저장/복원 실패(특히 앱 종료 직후) | 높음 | 높음 | 게임 진행, 점수, 난이도 상태 동기화 실패 시 진행 데이터 유실. |
| 난이도/시드 비결정성 | 중 | 중 | 랜덤 요소 재생성 및 재시작 조건이 안정적으로 고정되지 않으면 공정성 저하. |
| 타이머 정지/재개 및 백그라운드 진입 | 중 | 중 | 배경 진입 시 경과 시간 계산, 타임아웃 처리 검증 필요. |
| Undo/Redo 상태 반전 | 중 | 높음 | 이력 스택 인덱스 오프셋 및 애니메이션 동기화 이슈. |
| 광고 트리거가 게임 플로우를 가로채는 버그 | 중 | 중 | 중요 결정 타이밍(승/패 전환)에 광고 표시 유도 시 상태가 깨질 수 있음. |

## 4) 회귀 테스트 케이스
- [ ] TC-PUZ-01: 앱 종료 후 즉시 재진입해 진행 중 상태와 점수가 그대로 이어지는지 확인
- [ ] TC-PUZ-02: Undo/Redo 연속 수행 시 게임 보드가 예측 가능한 상태로 되돌아가는지 확인
- [ ] TC-PUZ-03: 타이머가 백그라운드 진입 및 복귀해도 경과 시간이 정확히 반영되는지 확인
- [ ] TC-PUZ-04: 난이도 변경/새 라운드 시작 후 동일한 시드 정책에서 테스트 데이터가 편향되지 않는지 확인
- [ ] TC-PUZ-05: 광고 로딩 실패/취소 상황에서 게임 진행이 중단되지 않는지 확인
